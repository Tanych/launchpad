Launchpad

A django based system that provides a stable URL for every item in the library's catalog. Various discovery services will link to these URLs. The page for each item will in turn link out to various other resources that provide methods for accessing the content of the items.

About the GW Findit Project

GW Library's Findit service connects you as quickly as we can to what you're looking for. It uses our catalog, journals, databases, and some additional tools we're mixing together here in Foggy Bottom. We're working to make it easier to get to what you're looking for or give you a way to get what you need even if it's not readily available at GW or our consortium peer libraries.

Limits on use

This system pulls data from several different sources. We and our colleagues in our consortium depend on these sources to keep our libraries running. If we find automated, heavy use of this service that negatively impacts the performance or availability of it or any of the related systems it uses, many people in our communities may be affected.

We reserve the right to block or limit access to this system by any users or automated agents that are impacting its performance or availability. If you are interested in bulk access to this data, or data like it, let us know directly. We'd be happy to discuss your interest and see if we can help.

Feature Summary

Here is a summary of some of the key elements of the Findit project user interface:

  • Summary information identifies each work (title, author, standard numbers)
  • Consortial holdings are listed, grouped by institution and library location
  • Real time availability -- based on live query of three different library systems
  • Associated records from all members libraries based on standard numbers (ISBN, ISSN, OCLC) are retrieved
  • Conditional logic for consortium borrowing is applied to the items found (for example a Reference book or an item Charged is not eligible)
  • Links to the Consortium loan request form and to an Interlibrary Loan Request form, as appropriate
  • Book jacket thumbnail images from OpenLibrary.org are displayed, when available
  • Non-Roman script (unicode for Chinese, Japanese, and other languages) is displayed whenever appropriate
  • An SMS link enables the patrons to text or email the item information to themselves
  • A standard URL pattern provides access from Worldcat, Google, and local catalogs based upon standard numbers (Bibliographic / Item ID, ISBN, ISSN, and OCLC)
  • Resources outside the consortium may be retrieved (via the GoogleBooks API and WorldCat API when the consortium does not have a match for a requested standard number)
  • The user interface uses CSS in order to follow the general look and feel of the library web site

GW Findit environment

The Findit project brings together information from three separate integrated library systems:

  • Georgetown University: Innovative Interfaces Millenium ILS
  • George Mason University: ExLibris Voyager ILS
  • Washington Research Library Consortium (WRLC): ExLibris Voyager ILS

The WRLC Voyager ILS is a union catalog. It is used to maintain bibliographic, holding, and item information by most members of the library consortium. These institutions are listed here with the approximate number of titles held by each:

  • American University (1,300,000)
  • Catholic University (860,000)
  • Gallaudet University (400,000)
  • George Washington University (1,700,000)
  • Howard University (760,000)
  • Marymount University (185,000)
  • University of the District of Columbia (575,000)

In addition, the WRLC Voyager ILS stores a copy of bibliographic information provided by two other local institutions that maintain their own ILS:

  • George Mason University (1,750,000)
  • Georgetown University (2,740,000)

The Voyager ILS hosted by WRLC does not have the holdings and items from the other two sytems. In order to obtain detailed holding and circulation status for George Mason and Georgetown it is necessary to query those systems directly.

The Django application framework follows the model-view-controller architectural pattern. The application optimizes the interactions with the integrated library systems by minimizing the number of database connections and employing caching mechanisms.

How Findit obtains real time availability

The Findit program runs in a Linux environment hosted at the WRLC facilities. Findit uses Oracle SQLPlus InstantClient software on this server to execute SQL queries against the WRLC Voyager Oracle database using a read-only user account. When a title is held by Georgetown or George Mason, Findit executes Z39.50 queries against the appropriate remote database. The application collates the responses and generates a web page or JSON data structure.

Findit API

Link patterns in Findit

The standard URL for any item in findit is its WRLC Voyager database bibliographic identifier, or "bib id". Every request to Findit should result in a URL like this:

http://findit.library.gwu.edu/item/3992408 In this example, 3992408 is the bib id.

This bookmark is stable, which is to say: if you want to bookmark it, please do. As long as this item is in our consortium catalog, this link will resolve to all the information we can provide you about the item and its availability.

There's an additional way you can create links to findit: using standard identifiers like ISBN and ISSN. The example above is a book, with ISBN 0439136350. ISBN and ISSN are very common, and it can be useful to create a link using these values. findit supports this kind of link:

http://findit.library.gwu.edu/isbn/0439136350

http://findit.library.gwu.edu/issn/0000-0019

There's one more identifier that redirects like this, and though it might be most useful to us librarians, you can use it too. It's for the OCLC number of an item:

http://findit.library.gwu.edu/oclc/00162792 It should work with a redirect to an /item/[bib id] link, just like ISBN and ISSN links.

Raw data from findit in JSON

We do our best to lay out the information on findit pages as clearly as we can, but maybe you just want the raw data under the hood. You can have it! Just add ".json" to the end of any /item/[bib id] URL:

http://findit.library.gwu.edu/item/3992408.json

We're using JSON because it's useful in most every programming language. You should be able to use your language of choice to process this data.

In each of these records, basic bibliographic data is at the top level, and separate holdings data is listed one by one. The data itself reflects several aspects of how we've managed data in libraries over many decades, so it might seemed odd if you've never seen anything like it before. It should mostly be self-explanatory, but if not, please don't hesitate to ask us about it.

We are actively refactoring / improving the source. The JSON structure is subject to change.

Indexing

Google site index

Content from the Findit project is gradually being index with the Google site index tool. This indexing is progressing at the rate of approximately 50,000 titles per day. There are somewhere in the neighborhood of 10 million titles in the WLRC collection. Searching a title and the phrase "gwu libraries" will yield search results that link to Findit by the item identifier.

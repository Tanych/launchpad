    About launchpad and the GW Findit Project

    Launchpad uses our catalog, journals, and databases to make it easier to get to what you're looking for and to give you a way to get what you need, even if it's not readily available at GW or our consortium peer libraries.

    Features

    • Standard URL patterns provide access from Worldcat, Google, and local catalogs based upon standard numbers for Item ID, ISBN, ISSN, and OCLC Number. [See Findit API, below]
    • Site pages are being indexed by Google; users may search in Google to locate library holdings.
    • Many titles held outside of the consortium are available via an Interlibrary Loan forms (via the GoogleBooks API and WorldCat API).
    • Links are provided to full text of books available from Open Library (based on LCCN lookup in the Open Library API).
    • Summary information is provided for each work (title, author, standard numbers).
    • All WRLC consortium holdings are listed for a selected title, grouped by institution and library location. Holdings available online are listed first.
    • Associated records from all members libraries based on standard numbers (ISBN, ISSN, OCLC) are retrieved.
    • Real time availability is displayed -- based on live query of three different library systems (WRLC, Georgetown, George Mason).
    • Conditional logic for consortium borrowing is applied to the items found (for example a Reference book or a Charged item is not eligible for consortium borrowing).
    • Links are provided to the Consortium loan request form and to an Interlibrary Loan Request form, as appropriate. The form fields are pre-populated by the Launchpad application.
    • Book jacket thumbnail images from OpenLibrary.org are displayed, when available.
    • Non-Roman characters (unicode for Chinese, Japanese, Arabic, and other languages) are displayed whenever appropriate.
    • Links are provided enabling the user to send item information as text or by email.
    • Online journal titles include the date range (as provided by the Serials Solutions 360 Link API).
    • Links to online full text, streaming video, or streaming audio content are displayed with the appropriate labels (Full Text Online, Watch Online, or Listen Online).
    • The software is available is free software under an MIT-style license (see below).
    • The fonts, colors, and other style elements as well as certain behaviors -- such as the sorting rules -- are configurable.

    GW Findit environment

    We build launchpad at GW Libraries to bring together information from three separate integrated library systems:

    • Washington Research Library Consortium (WRLC): ExLibris Voyager ILS
    • Georgetown University: Innovative Interfaces Millenium ILS
    • George Mason University: ExLibris Voyager ILS

    The WRLC Voyager ILS is a union catalog. It is used to maintain bibliographic, holding, and item information by most members of the library consortium. These institutions are listed here with the approximate number of titles held by each:

    • American University (1,300,000)
    • Catholic University (860,000)
    • Gallaudet University (400,000)
    • George Washington University (1,700,000)
    • Howard University (760,000)
    • Marymount University (185,000)
    • University of the District of Columbia (575,000)

    In addition, the WRLC Voyager ILS stores a copy of bibliographic information provided by two other local institutions that maintain their own ILS:

    • George Mason University (1,750,000)
    • Georgetown University (2,740,000)

    The Voyager ILS hosted by WRLC does not have the holdings and items from the other two sytems. In order to obtain detailed holding and circulation status for George Mason and Georgetown it is necessary to query those systems directly.

    The Findit programming environment

    The Findit project is Python and Django based application. It assumes the presence of a primary Voyager/Oracle database, with direct read-only access using Oracle's SQL Plus client. At present, it has no separate application database of its own, and it does not write data to Oracle.

    We recommend developing, testing, and deploying launchpad in a virtualenv. Instructions and requirements are in the README.

    Launchpad optimizes the interactions with the integrated library systems by minimizing the number of database roundtrips with direct SQL queries. We run it in production at GW Libraries with memcached page caching (as provided by django) enabled.

    How Findit obtains real time availability

    The Findit program runs in a Linux environment hosted at the WRLC facilities. Findit uses Oracle SQLPlus InstantClient software on this server to execute SQL queries against the WRLC Voyager Oracle database using a read-only user account. When a title is held by Georgetown or George Mason, Findit executes Z39.50 queries against the appropriate remote database. The application collates the responses and generates a web page or JSON data structure.

    Findit API

    Link patterns in Findit

    The standard URL for any item in findit is its WRLC Voyager database bibliographic identifier, or "bib id". Every request to Findit should result in a URL like this:

    http://findit.library.gwu.edu/item/3992408 In this example, 3992408 is the bib id.

    Standard identifiers like ISBN, ISSN, and OCLC also have link patterns

    http://findit.library.gwu.edu/isbn/0439136350

    http://findit.library.gwu.edu/issn/0000-0019

    http://findit.library.gwu.edu/oclc/00162792

    Data Raw data from findit in JSON

    Just add ".json" to the end of any /item/[bib id] URL:

    http://findit.library.gwu.edu/item/3992408.json

    In the JSON, basic bibliographic data is at the top level, and separate holdings data is listed one by one.

    We are actively refactoring / improving the source. The JSON structure is subject to change.

    Indexing

    Schema.org

    The Findit pages contain semantic markup compatible with standards established by schema.org.

    Google site index

    Content from the Findit project is gradually being index with the Google site index tool. This indexing is progressing at the rate of a few thousand titles per day. There are somewhere in the neighborhood of 10 million titles in the WLRC collection. Searching a title and the phrase "gwu libraries" will yield search results that link to Findit by the item identifier.

    For example search edward tufte gwu libraries

    Enabling the configuration setting ENABLE_SITEMAPS will expose all the non-suppressed item URLs in the catalog to crawlers using the sitemaps specification.
